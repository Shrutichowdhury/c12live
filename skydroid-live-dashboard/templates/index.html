Quick-fill:
Steps to get real gimbal control working
  1. Download the skydroid-live-dashboard/ folder to a PC on the same network as the camera.
  2. Double-click run_local.bat (Windows) or run ./run_local.sh (Mac/Linux) — installs Python deps and starts the server automatically.
  3. Open http://localhost:5000 in your browser.
  4. Click Test — if green, UDP port 5000 is reachable (confirmed from official SkyDroid SDK).
  5. Click Multi-Protocol Scan — sends real Skydroid ASCII commands and shows any replies. Watch if the gimbal physically moves when the Center command fires.
  6. Use the Raw Command Terminal above to send any individual command and see the exact bytes exchanged.
  7. Click Enable Real Control — all D-pad, zoom, and camera buttons will now send real ASCII commands directly to the camera over UDP.

Protocol confirmed from official SkyDroid SDK AAR v1.9.1 decompile.
Camera port: 5000 UDP (SDK: getUDPPayload(C12, 5000, "192.168.144.108", 5000)).
Command format: #TPUG2wPTZ01 + 2-hex-digit checksum → #TPUG2wPTZ016B.
Thermal palette: #TPUD2wIMG00...#TPUD2wIMG09 (0=White Hot, 9=Glory Hot).
Discrete zoom: #TPUD2wDZM00...#TPUD2wDZM04 (0=original, 1-4=digital zoom).
Time sync: #TPUDFwTIM<HHmmss><ddMMyy>.00 + CRC.
